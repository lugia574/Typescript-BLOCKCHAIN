# 4 CLASSES AND INTERFACES

## 4.0 Classes

이번에는 ts로 객체지향 프로그래밍을 해볼꺼임

ts 는 class 생성자를 만들때 this 안써도 됨

```ts
class Player1 {
  constructor(
    private firstName: String,
    private lastName: String,
    public nickName: String
  ) {}
}
```

이러면 js 파일로 컴파일러 하면

```js
class Player1 {
  constructor(firstName, lastName, nickName) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.nickName = nickName;
  }
}
```

이렇게 됨 js 는 private 같은게 없음

그래서 다 접근해서 후루룩 해버리는데

그걸 ts가 사전에 막아줌

그래서

```ts
//ts

const ex = new Player1("ex", "las", "엑스");

ex.firstName; //ERROR
```

ts 에서는 firstName 속성은 private 여서 Player1 클래스 내에서만 액세스 할 수 잇다고 막음

근데 js 에서는 그런거 없음

이처럼 js에서 못하는 걸 ts에서는 할 수 있다~

---

다음으로는 추상클래스임

추상클래스는

    다른 클래스가 상속받을 수 있는 클래스

    긍데 이 클래스는 직접 새로운 인스턴스를 만들 수 없어

    한마디로

    클래스를 상속해서 사용하려는 놈에게 특정기능은 있어야한다 하는거임

```ts
abstract class User {
  constructor(
    private firstName: String,
    private lastName: String,
    public nickName: string
  ) {}
}

class Player1 extends User {}

const ex = new Player1("ex", "las", "엑스");
```

여기서 메소드도 넣어보자

```ts
abstract class User {
  constructor() {} // 생략....

  getFullName() {
    return `${this.firstName}  ${this.lastName}`;
  }
}
```

이 소드를 private 하거나 다른 접근제어자를 써서

접근을 제한 할 수 있음

물론 js 에서는 그런거 없음 ㅋ

private를 사용하면 상속받은 클래스 안에서 마저도 this 사용해 접근 불가능

그래서 protected를 사용하면 상속받은 클래스 안에서 this 사용해 접근 가능

물론 protected로 지정된 것들은 외부에서 사용이 불가능

추상클래스 안에 추상 메소드는 call signature만 적어야 함

```ts
abstract class User {
  abstract getFullName(): void;
}
```

    추상메소드는 내가 추상 클래스를 상속받는다면 꼭 구현 해야하는 메소드

여기서 상속받은 자식 클래스가 firstName 같은 변수, 메소드에 접근 하게 하려면

public이나 protected를 쓰는게 맞음

```ts
abstract class User {
  constructor(
    protected firstName: String,
    protected lastName: String,
    protected nickName: string
  ) {}
}
```

| 구분　　  | 　선언한 클래스 내　 | 상속받은 클래스 내　 | 인스턴스 |
| :-------: | :------------------: | :------------------: | :------: |
|  private  |          ⭕          |          ❌          |    ❌    |
| protected |          ⭕          |          ⭕          |    ❌    |
|  public   |          ⭕          |          ⭕          |    ⭕    |

## 4.1 Recap

생략

할려고 했는데 안본 걸 쓰네

사전을 구현 해본데

사전이 words 오브젝트를 가진다고 알려줄꺼임

```ts
class Dict2 {
  constructor(private x: string) {}
}
```

원래는 이런식으로 해야할텐데 이렇게 안할꺼래

constructor에서 값을 형태를 지정 안할꺼래

```ts
type Words = {
  [key: string]: string;
};

class Dict {
  private words: Words;
}
```

private words 를 선언해주고 words의 타입을 알려줄꺼임

오브젝트 타입 설정할때 이렇게도 씀

key 값은 무조건 string 인거임

근데 이러면 Dict 클래스에 words 에 에러가 뜨는데

words는 initializer 가 없고

constructor에서 정의된 sign 이 아니라고 ㅈㄹ함

```ts
type Words = {
  [key: string]: string;
};

class Dict {
  private words: Words;

  constructor() {
    this.words = {};
  }
}
```

```ts
// dictionary 만들기

type Words = {
  [key: string]: string;
};

class Dict {
  private words: Words;

  constructor() {
    this.words = {};
  }

  // 단어 추가 메소드
  add(word: Word) {
    if (this.words[word.term] === undefined) {
      // undefined 는 사전에 없는가를 묻는거
      this.words[word.term] = word.def;
    }
  }

  // 단어 보여주기

  def(term: string) {
    return this.words[term];
  }
}

class Word {
  constructor(public term: string, public def: string) {}
}

// 객체 생성
const fucking = new Word("퍽킹", "엿머겅");

const dict = new Dict();

dict.add(fucking);

dict.def("퍽킹");
```

```js
"use strict";
// dictionary 만들기
class Dict {
  constructor() {
    this.words = {};
  }
  // 단어 추가 메소드
  add(word) {
    if (this.words[word.term] === undefined) {
      // undefined 는 사전에 없는가를 묻는거
      this.words[word.term] = word.def;
    }
  }
  // 단어 보여주기
  def(term) {
    return this.words[term];
  }
}
class Word {
  constructor(term, def) {
    this.term = term;
    this.def = def;
  }
}
// 객체 생성
const fucking = new Word("퍽킹", "엿머겅");
const dict = new Dict();
dict.add(fucking);
dict.def("퍽킹");
```

## 4.2 Interfaces

Typescript에서 type은 내가 원하는 대로 타입을 정의해서 할 수 있음

```ts
type Player = {
  nickname: string;
  healthBar: number;
  // ...~
};
```

이렇게 player 오브젝트의 모양을 알려주고 있는 거임

```ts
const p1: Player = {
  nickname: "hi",
  healthBar: 99,
};
```

그리고 이렇게 값을 만들 수 있음

타입을 사용하는 다른 방법으로는

```ts
type Food = string;

const kim: Food = "김김김";
```

```ts
type Nickname = string;
type healthBar = number;
type Friends = Array<string>;

type Player = {
  nickname: Nickname;
  healthBar: healthBar;
  friends: Friends;
};
```

이렇게 가능함

또한 타입을 지정한 옵션만으로 제한 할 수 도 있음

```ts
type Team = "red" | "blue" | "yellow";

type Player = {
  nickname: string;
  team: Team;
};
```

이런식으로도 가능함

이제 오브젝트 모양을 설명하는 다른 방법인 인터페이스에 대해 알아볼꺼임

인터페이스는 타입이랑 비슷함

```ts
interface Player {
  nickname: Nickname;
  healthBar: healthBar;
  friends: Friends;
}
```

타입은 여러가지 타입을 설정할 수 있지만

```ts
interface fjk = string; //ERROR
```

이렇게는 안됨

인터페이스는 오직 오브젝트의 모양을 특정해주기 위한거임

한마디로 타입이 좀 더 많은걸 할 수 있음

하지만 인터페이스는 클래스와 비슷하게 사용할 수 있음

인터페이스는 상속이 가능함

```ts
interface User {
  name: string;
}

interface Player extends User {}

const nico: Player = {
  name: "ssss",
};
```

이렇게 부모 인터페이스의 멘버를 그대로 상속 받아서 사용 쌉가능

타입으로 이렇게 할려면

```ts
type User = {
  name: string;
};

type Player = User & {};

const nico: Player = {
  name: "ssss",
};
```

이럼

인터페이스의 또 다른 특징으로 property 를 축적할수 있음

```ts
interface User {
  name: string;
}
interface User {
  lastname: string;
}
interface User {
  healthBar: number;
}

const nico: User = {
  name: "nico",
  lastname: "sss",
  healthBar: 99,
};
```

이렇게 가능함

## 4.3 Interfaces part Two

추상클래스는 다른 클래스가 가져야할 property랑 메소드를 명시 할 수 있도록 도와줌

```ts
abstract class USer {
  constructor(protected firstName: string, protected lastName: string) {}

  abstract sayHi(name: string): string;
  abstract fullName(): string;
}
```

이렇게 추상 클래스를 하면

```ts
class Player2 extends User2 {
  fullName() {
    return `${this.firstName} ${this.lastName}`;
  }
  sayHi(name: string) {
    return `${name} Hi.. hahaha.. my name is ${this.fullName()}`;
  }
}
```

이렇게 구현이 되겠지

추상클래스의 문제점은 자바스크립트에는 추상화의 개념이 없어

```js
class User2 {
  constructor(firstName, lastName) {
    this.firstName = firstName;
    this.lastName = lastName;
  }
}
```

이렇게 추상클래스가 js 로 컴파일되면 그냥 일반 클래스로 변해버림

쓰지도 않을 클래스로 남아서 자리 차지 하고 있는거임

이럴때 인터페이스를 써야하는 법임

인터페이스는 가벼움

인터페이스를 컴파일 하면 아예 사라져버림

인터페이스를 쓰면 이렇게 됨

```ts
interface User2 {
  firstName: string;
  lastName: string;

  sayHi(name: string): string;
  fullName(): string;
}

class Player2 implements User2 {
  constructor(public firstName: string, public lastName: string) {}

  fullName() {
    return `${this.firstName} ${this.lastName}`;
  }
  sayHi(name: string) {
    return `${name} Hi.. hahaha.. my name is ${this.fullName()}`;
  }
}
```

여기서 문제가 2개 있는데

하난 인터페이스는 private property 를 못쓴다는거임

private를 쓰면 에러떠서 public으로 선언했자너

또 다른건 생성자(constructor)를 선언 부분에서 할 수 없어서

클래스 구현할때에 박아줘야한다는 점임
